# mayak-digit: Цифровой словарь Маяковского

![Python](https://img.shields.io/badge/python-3.12-blue.svg)
![Status](https://img.shields.io/badge/status-active-success.svg)

**MayakDigit (Mayak-2D)** — это цифровой проект по созданию полноценного кроссжанрового авторского словаря В. В. Маяковского, совмещающий в себе инструменты цифровой филологии и NLP.

В данный момент проект существует в статусе прототипа.

---

## 🚀 Основные функции

* **Индекс Маяка (Proximity Index):** Глобальный расчет контекстуальной близости слов с учетом переносов строки и границ предложений.
* **Семантический компаратив (Navec):** Сравнение авторского словоупотребления с общеязыковой нормой. Система находит синонимы в модели Navec и разделяет их на:
    * *Используемые автором* (с расчетом их личного контекстного веса).
    * *Проигнорированные автором* (семантические лакуны).
* **Умная лемматизация и фильтрация:** Предобработка текстов с использованием морфологического анализа для отсечения дублей и нормализации выдачи синонимов.
* **Сравнение периодов (Period Comparison):** Анализ развития авторского словоупотребления — сравнение индексов контекстуальной близости слова в разные периоды творчества (дореволюционный vs 1920-е и т.д.) с визуализацией дельта-метрик.
* **Визуализация контекстуальных кластеров:** Семантическая карта корпуса на основе Индекса Маяка — UMAP-редукция матрицы контекстуальных весов и K-Means кластеризация топ-300 слов. Интерактивный Plotly-граф с цветными кластерами и поиском по словам.
* **Статистика корпуса:** Общие метрики (количество текстов, токенов, лемм), частотный словарь с фильтрацией стоп-слов, динамика роста словарного запаса по годам, Type-Token Ratio. Векторная карта частотных слов на основе PCA (Navec).
* **Неологизмы (beta):** Анализ hapax legomena — слов, встречающихся только в корпусе Маяковского и отсутствующих у его современников (1900–1930). Фильтрация однократных единиц, проверка наличия в Navec, экспорт в CSV.

Для создания поэтического корпуса использованы предварительно обработанные данные из цифрового корпуса PoeTree (https://versologie.cz/poetree/browser/author?corpus=ru&id_author=18)

Cсылка на скачивание полного корпуса стихотворений: https://drive.google.com/drive/folders/1zskVJAW-01XhPkRfT9vcxAoorS3gIlcT?usp=sharing

Для создания публицистического корпуса использованы материалы Полного собрания сочинений в 13 томах (М.: Гослитиздат, 1955-1959.) (публицистический корпус находится в разработке)

Для сопоставления метрик использован предварительно обработанный дамп поэтических текстов из корпуса PoeTree.

## ⚙️ Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/yourusername/mayak-digit.git
   cd mayak-digit
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Подготовьте модели:**
   Скачайте предобученную модель Navec (рекомендуется hudlit) и поместите файл .tar в директорию models/

4. **Настройте переменные окружения (для LLM-интерпретации):**
   Создайте файл `.env` в корне проекта:
   ```
   DEEPSEEK_API_KEY=your_key_here
   ```

5. **Запустите приложение:**
   ```bash
   streamlit run app/app.py
   ```

6. **(Опционально) Генерация семантической карты кластеров:**
   Если файл `data/word_cluster_map.json` отсутствует, его можно сгенерировать из корня проекта. Также можно использовать уже предобработанную карту.
   ```bash
   python -m src.map_builder
   ```
   Это занимает 20–30 минут в первый раз (300 слов × 20 кластеров).

## Методология: Как считается "Вес"?

Для каждого вхождения искомого слова (таргета) система сканирует всё произведение целиком и в отличие от стандартных методов окон (window size) учитывает глобальный контекст.

Итоговый вес слова-соседа вычисляется по формуле затухания:

$$Weight = (decay_{dist}^{d}) \times (decay_{brk}^{n\_brks}) \times (decay_{sent}^{n\_sents})$$

Где:
* **$d$** — физическое расстояние в словах (чем дальше, тем меньше связь).
* **$n\_brks$** — количество переносов строк («лесенка»). Каждый разрыв уменьшает семантическую близость.
* **$n\_sents$** — количество границ предложений. Переход в другое предложение считается сильным барьером.

Модель автоматически суммирует веса от всех вхождений слова в тексте, позволяя оценить контекстуальные пересечения более точно.

### Семантическая карта кластеров

На основе матрицы контекстуальных весов Индекса Маяка строится семантическая карта корпуса:

1. Для топ-300 слов вычисляется вектор весов ко всем остальным словам → матрица 300×300.
2. UMAP редуцирует пространство до 2D, сохраняя локальную структуру.
3. K-Means разбивает слова на 20 кластеров по близости контекстуальных профилей.

Результат — интерактивная карта, где расстояние между словами отражает их контекстуальную близость в поэзии Маяковского, а не в общем языке.
